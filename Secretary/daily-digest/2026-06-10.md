### cronとPATH管理
**概要**: cronジョブでは `/Applications/cmux.app/Contents/Resources/bin` 等のユーザー固有PATHが引き継がれないため、`claude` コマンドが exit 127 で無音終了していた。スクリプト冒頭に `export PATH=...` を明示することで解決。  
**なぜ深めたいか**: cronの自動化は今後も頻繁に使う。「本番で動かない」を事前に防ぐためのデバッグパターンとして汎用性が高い。  
**推奨調査キーワード**: `cron PATH environment variable best practice`, `macOS launchd environment variables`, `bash set -e error handling cron`

### `set -e` の落とし穴と安全なエラーハンドリング
**概要**: `set -e` は任意コマンドの失敗で即スクリプト終了させるが、コマンド置換（`$(...)` ）の失敗でも同様に動作し、ログも残らずサイレント終了する。`|| true` や明示的エラーキャプチャが必要。  
**なぜ深めたいか**: 自動化スクリプトで最もやらかしやすい落とし穴。ログがないと原因特定に時間がかかる。  
**推奨調査キーワード**: `bash set -e command substitution pitfall`, `bash error handling best practices`, `bash trap ERR logging`

### [Tailwind v4 + カスタムCSSの競合]
**概要**: `globals.css` にカスタムCSSクラスを書いたが、Tailwind v4の処理系（Turbopack）がキャッシュや`@layer`外のスタイルを無視し、CSSがブラウザに届かない問題が発生した。サーバー再起動で解決。  
**なぜ深めたいか**: Tailwind v4はv3と内部処理が大きく変わっており、カスタムCSSの書き方・`@layer`の使い方・Turbopackとの相性を正しく理解しないと同じ問題が繰り返す。  
**推奨調査キーワード**:
- `Tailwind CSS v4 custom CSS @layer compatibility`
- `Turbopack CSS cache invalidation not working`
- `Tailwind v4 globals.css keyframes`

### [CSSの `box-sizing` 差異によるピクセルズレ]
**概要**: HugMugはWordPress（デフォルト `content-box`）、cosme-zemiはNext.js + Tailwind（グローバルに `border-box`）。同じ `width: 50px` でも実際のボックスサイズが変わり、二重ピルバッジの影のオフセットがズレた。  
**なぜ深めたいか**: サイトリバースエンジニアリング時の「見た目は合ってるのにズレる」系バグの典型原因。デザイン踏襲タスクで必ず踏む落とし穴として体系化したい。  
**推奨調査キーワード**:
- `box-sizing border-box vs content-box CSS pseudo-elements`
- `Tailwind CSS resets box-sizing impact`
- `CSS ::before ::after width calculation border-box`

### Playwright でのLazy Load 完全対応

**概要**: hugmug.jp のスクリーンショット取得で、スクロールしても画像が読み込まれない問題に直面。`loading="eager"` への強制変換・`data-src` 解除・スクロール後 networkidle 待ちの組み合わせで対処した。

**なぜ深めたいか**: webスクレイピング・参照サイト収集のたびに同じ問題にぶつかる。パターンとして持っておくと再現実装が速くなる。

**推奨調査キーワード**: `playwright lazy load images networkidle`, `playwright scroll until images loaded`, `playwright force eager loading`

### CSS変数・CSSクラスによる動的テーマの解析

**概要**: hugmug のカラーが `color_01〜XX` クラスで切り替わる仕組みで、ヘッドレス環境での RGB 自動判定が失敗。CSS変数や SVG 経由で色が適用されている場合、`page.evaluate()` で computed color を取っても `rgb(0,0,0)` が返る。

**なぜ深めたいか**: デザインリサーチ時のカラースキーム自動抽出に応用できる。CSS変数の解決方法を理解しておくと今後の参照サイト解析が正確になる。

**推奨調査キーワード**: `playwright get CSS variable value`, `playwright computed style CSS custom properties`, `playwright getComputedStyle CSS variables`

### cron/バッチの UTC・タイムゾーン問題

**概要**: Claude Routines（`date +%u`）が UTC 基準で曜日を返すため、JST 7:00 実行でも前日の曜日テーマが送られるバグを修正。`TZ=Asia/Tokyo date +%u` で解決。

**なぜ深めたいか**: 自動配信・バッチ・スケジューラーを複数本運用するにあたって同じ問題が再発しやすい。サーバーサイドの日時処理の「UTC=デフォルト」は常に意識すべき設計原則。

**推奨調査キーワード**: `cron job timezone JST UTC difference`, `bash date TZ environment variable`, `cloud scheduler timezone aware cron`

📚 今日の学び｜エンジニアリング/アーキテクチャ

• AIコード生成が本番コードの41%を占める時代へ：2025年時点で全コードの約41%がAI生成となり、高AI採用組織では2026年後半に50%超の見込み。McKinseyによると一部組織でコード生成速度30〜50%向上、GitHub Copilotの有料サブスクライバーは2026年1月に470万人（YoY 75%増）に達した。
　🔗 https://modall.ca/blog/ai-in-software-development-trends-statistics

• AI生成コードの45%のテストでセキュリティ脆弱性を検出：Veracodeの2025年 GenAI Code Security Reportで、100以上のLLMが生成したコードの45%のテストでリスクのある脆弱性が発見された。すでに20%の組織が本番でのAI起因のセキュリティ侵害を経験しており、コードレビューにセキュリティ特化フローの組み込みが急務。
　🔗 https://dev.to/austinwdigital/ai-assisted-development-in-2026-best-practices-real-risks-and-the-new-bar-for-engineers-3fom

• WebAssembly（WASM）がサーバーサイドで主流化：W3CがWasm 3.0を2025年9月に正式標準として採択し、AWS・Google Cloud・Azureが主流オプションとしてWasmベースのサーバーレス関数を提供開始。従来コンテナのコールドスタート（100〜500ms）に対しWasmは5ms未満を実現し、Figma・Google・Shopify・Adobeが本番環境で大規模運用中。
　🔗 https://codefinity.com/blog/WebAssembly-Beyond-the-Browser:-Edge-and-Server-Side-WASM-in-2026

• プラットフォームエンジニアリングがML・データエンジニアにも拡大：2026年、対象ユーザーはアプリ開発者だけでなくデータエンジニア・MLエンジニア・アナリストにまで拡大。AI駆動のアーキテクチャ最適化により、コストとレイテンシの目標に応じてシステムを人手なしに動的に再設計するプラットフォームが登場し始めている。
　🔗 https://leanopstech.com/blog/platform-engineering-trends-2026/

• DX Core 4でエンジニアリング生産性を4軸計測：Intercomが20%向上：速度・有効性・品質・インパクトの4軸からなるDX Core 4を300社超（テック・金融・小売・製薬）が導入。Q1 2026データではAI生成コードが本番の平均27.4%を占め、早期採用チームはPRスループットで約7.8%向上。Intercomは生産性20%改善・R&D機能開発時間14%増を達成。
　🔗 https://getdx.com/research/measuring-developer-productivity-with-the-dx-core-4/

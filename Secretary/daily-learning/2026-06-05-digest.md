🔍 会話から深掘り｜2026-06-05

**MCPツールのサブエージェントスコーピング**
• サブエージェントはデフォルトでメイン会話のすべての内部ツールとMCPツール（lazyweb等）を継承するが、`tools` フィールド（allowlist）で明示指定した場合は指定外のMCPツールは引き継がれない。これがサブエージェントにlazyweb等のMCPツールが渡らなかった原因。
• `disallowedTools` フィールドで特定ツールのみを除外するdenylist方式も使える。両方設定した場合は `disallowedTools` が先に適用される。`mcpServers` フィールドでサブエージェントに使わせるMCPサーバーを明示指定することも可能。
• サブエージェントはさらに別のサブエージェントをスポーンできない（無限ネスト防止）。`Agent` / `AskUserQuestion` / `ScheduleWakeup` 等のUI依存ツールも利用不可。プラグイン経由のサブエージェントはセキュリティ上の理由から `mcpServers` / `hooks` / `permissionMode` フィールドをサポートしない。

**LIFF審査中のローカル開発バイパスパターン**
• LINE公式の `@line/liff-mock` パッケージを使うと、LIFF SDKにモックモードを追加できる。`npm install @line/liff-mock` でインストール後、`liff.use(new LiffMockPlugin())` を登録し、`liff.init({ liffId: 'liff-xxxx', mock: true })` でLINEサーバーへの接続なしに動作可能になる。
• `liff.$mock.set()` で `getProfile()` 等のLIFF APIの戻り値を自由に設定でき、`liff.$mock.clear()` でデフォルトに戻せる。`NEXT_PUBLIC_USE_MOCK=true` 環境変数による独自実装とは別の、LINEの公式モック手段として活用できる。
• 2026年1月から日本の新規LINE MINIアプリは「チャンネル同意簡略化」が必須化。2026年3月からAndroid版LINEのedge-to-edge対応により、LIFFアプリ下部ボタンがナビゲーションバーと重なる可能性があるため、パディング調整が必要。

**カードUIのアスペクト比設計原則**
• ヒーローカード（詳細ページ上部・単体強調表示）では16:9が最適。カード幅全体に広がる横長画像がコンテンツの第一印象を担い、情報密度より視覚的インパクトを優先する場面に適している。正方形画像は「切手」状態になり、画面幅に対して視覚的な抜け感が出てしまう。
• 一覧カードのサムネイルは1:1（正方形）が視覚的整合性を保ちやすく、テキスト情報との比率バランスも取りやすい。Netflixは大型サムネイルのA/Bテストで、小サムネイル＋テキストよりも大きな画像を好むユーザーが多いことを実証している。
• デザインシステムの原則は「コンテキストごとに比率を定義」。ヒーロー：16:9、カードグリッド：3:2または4:3、アバター・一覧サムネイル：1:1、という使い分けが一般的。カード内テキストは最大100文字・3行以内に収めることでスキャナビリティを確保する。
